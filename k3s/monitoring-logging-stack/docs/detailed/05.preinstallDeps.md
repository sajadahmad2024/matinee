## 📊 4. kube-state-metrics Setup

To collect cluster-level metrics such as node status, pod count, resource usage, etc., we use **kube-state-metrics** — a service that listens to the Kubernetes API and exposes metrics in Prometheus format.

A helper script named `kube-metrics-setup.sh` is provided to simplify the setup using Helm.

### 🛠️ Script Purpose

The script does the following:

1. Adds the official Prometheus Helm chart repo (if not already added).
2. Updates Helm repositories.
3. Ensures the `kube-system` namespace exists.
4. Installs the `kube-state-metrics` Helm chart in the `kube-system` namespace **only if** it hasn’t already been installed.

### 📂 Script Location

``` 
./kube-metrics-setup.sh

```

### ▶️ One-Time Execution

Run this script **once** after setting up your Kubernetes cluster:

```bash 
chmod +x kube-metrics-setup.sh
./kube-metrics-setup.sh

```

> ⚠️ Note: The script is idempotent. If the chart is already installed, it will not reinstall or modify anything.
