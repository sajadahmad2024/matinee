## Grafana Dashboards Management

### Directory Structure

```bash
grafana/
├── dashboards
│   ├── create-dashboards.sh               # Script to create ConfigMaps from dashboard JSONs
│   ├── global-dashboard-definition.yml    # ConfigMap defining dashboard providers
│   └── raw-dashboards-files/              # JSON files of individual Grafana dashboards
│       ├── backend-prod-apm-dashboard.json
│       ├── blackbox-exporter-dashboard.json
│       ├── kube-state-metrics-dashboard.json
│       └── node-exporter-dashboard.json
├── datasources
│   └── datasources-configmap.yml          # ConfigMap for Grafana datasource
└── manifest.yml                           # Main Kubernetes manifest for Grafana

```

---

### Prerequisite

Before deploying the Grafana pod, **you must run the `create-dashboards.sh` script** to create dashboard ConfigMaps.

```bash
cd grafana/dashboards
./create-dashboards.sh 
```

This script contains the following `kubectl` commands:

```bash

## Node Exporter dashboard
kubectl create configmap grafana-dashboard-node-exporter \
  --from-file=raw-dashboards-files/node-exporter-dashboard.json -n monitoring

## Kube state metrics dashboard
kubectl create configmap grafana-dashboard-kube-state-metrics \
  --from-file=raw-dashboards-files/kube-state-metrics-dashboard.json -n monitoring

## Blackbox exporter dashboard
kubectl create configmap grafana-dashboard-blackbox-exporter \
  --from-file=raw-dashboards-files/blackbox-exporter-dashboard.json -n monitoring

## APM dashboard
kubectl create configmap grafana-dashboard-backend-prod \
  --from-file=raw-dashboards-files/backend-prod-apm-dashboard.json -n monitoring

```

### ➕ Adding a New Dashboard

To add a new dashboard:

1. **Place the dashboard JSON file** in the `raw-dashboards-files/` directory.
2. **Append a new `kubectl create configmap` command** to `create-dashboards.sh` with a unique name.
3. **Update `manifest.yml`** with a new volume and volumeMount section.
4. **Update `global-dashboard-definition.yml`** (`grafana-dashboards-definition` ConfigMap) with a new dashboard provider entry.

---

### Grafana `manifest.yml` Volume Configuration

Update the `grafana` deployment manifest with the correct volumes and mounts:

### `volumeMounts` section:

```yaml

volumeMounts:
  - name: datasources
    mountPath: /etc/grafana/provisioning/datasources
  - name: dashboards
    mountPath: /etc/grafana/provisioning/dashboards
  - name: dashboard-node-exporter
    mountPath: /var/lib/grafana/dashboards/kubernetes/node-exporter
  - name: dashboard-kube-state-metrics
    mountPath: /var/lib/grafana/dashboards/kubernetes/kube-state-metrics
  - name: dashboard-blackbox-exporter
    mountPath: /var/lib/grafana/dashboards/kubernetes/blackbox-exporter
  - name: apm-backend-dashboard
    mountPath: /var/lib/grafana/dashboards/custom/backend-apm

```

### `volumes` section:

```yaml

volumes:
  - name: datasources
    configMap:
      name: grafana-datasource
  - name: dashboards
    configMap:
      name: grafana-dashboards-definition
  - name: dashboard-node-exporter
    configMap:
      name: grafana-dashboard-node-exporter
  - name: dashboard-kube-state-metrics
    configMap:
      name: grafana-dashboard-kube-state-metrics
  - name: dashboard-blackbox-exporter
    configMap:
      name: grafana-dashboard-blackbox-exporter
  - name: apm-backend-dashboard
    configMap:
      name: grafana-dashboard-backend-prod

```

### Global Dashboard Definition

This is defined in the ConfigMap `grafana-dashboards-definition` (file: `global-dashboard-definition.yml`).

Each dashboard should have its own `provider` entry. Example:

```yaml

apiVersion: 1
providers:
  - name: 'node-exporter'
    orgId: 1
    folder: ''
    type: file
    disableDeletion: false
    editable: true
    options:
      path: /var/lib/grafana/dashboards/kubernetes/node-exporter

  - name: 'kube-state-metrics'
    orgId: 1
    folder: ''
    type: file
    disableDeletion: false
    editable: true
    options:
      path: /var/lib/grafana/dashboards/kubernetes/kube-state-metrics

  - name: 'blackbox-exporter'
    orgId: 1
    folder: ''
    type: file
    disableDeletion: false
    editable: true
    options:
      path: /var/lib/grafana/dashboards/kubernetes/blackbox-exporter

  - name: 'backend-apm'
    orgId: 1
    folder: ''
    type: file
    disableDeletion: false
    editable: true
    options:
      path: /var/lib/grafana/dashboards/custom/backend-apm

```

### Summary

| Task | Where to update |
| --- | --- |
| Add dashboard JSON | `raw-dashboards-files/` |
| Create ConfigMap | `create-dashboards.sh` |
| Mount dashboard into Grafana | `volumeMounts` & `volumes` in `manifest.yml` |
| Define dashboard provider | `global-dashboard-definition.yml` |

### Note
While adding new dashboards please add your custom/application related dashboards to custom folder. And kubernetes related document to the kubernetes folder only.
