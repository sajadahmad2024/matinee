## Node Exporter - System Metrics Daemon

### Purpose

The **Node Exporter** is deployed as a **DaemonSet** to collect **host-level metrics** from every node in the cluster. It exposes CPU, memory, disk, and other hardware/system statistics at port `9100`, enabling Prometheus to scrape metrics for system-level monitoring.

### Key Configuration

- **Runs on every node** using a `DaemonSet`
- Uses `hostPID: true` and `hostNetwork: true` to access node-level information
- Mounts key directories (`/proc`, `/sys`, `/`) as read-only
- Binds to `hostPort: 9100` to expose metrics
- Uses a `ServiceAccount` named `node-exporter`
- Exposes itself via a Kubernetes `Service` for Prometheus scraping

### Do Not Modify

- Do **not change** the volume mounts or `-path.*` arguments — they are required for accurate metrics.
- The `hostPort` is necessary for node-level accessibility.
- Avoid updating the image to `latest` manually unless a tested version is required (consider pinning to a specific version for stability).

### Security & RBAC

- Ensure your Prometheus is allowed to scrape from the `node-exporter` service.
- This setup assumes monitoring nodes are labeled and accessible appropriately.
