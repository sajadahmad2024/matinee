## Monitoring Namespace

The `monitoring` namespace is a **dedicated Kubernetes namespace** used to isolate all monitoring-related components and resources, such as:

- Prometheus
- Grafana
- Loki
- Alertmanager
- kube-state-metrics
- node-exporter
- blackbox-exporter
- Persistent Volume Claims for monitoring data

### Best Practices

- **Keep all monitoring components in this namespace** to simplify RBAC, resource isolation, and lifecycle management.
- Apply namespace-specific resource quotas or limits if needed.
- Use `kubectl get all -n monitoring` for debugging or checking component status.

> 💡 Do not deploy unrelated apps into this namespace. It’s meant solely for observability tools and data.
