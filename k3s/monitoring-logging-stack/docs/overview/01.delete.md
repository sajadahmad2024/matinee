## Step 1: Delete the `monitoring` Namespace

All monitoring components are deployed inside the `monitoring` namespace. To remove them:

```bash
kubectl delete ns monitoring
```

This will delete:

- All deployments (Prometheus, Grafana, Loki, etc.)
- ConfigMaps, Services, Secrets, and other resources
- Any volume claims associated with the namespace

---

## ⚠️ Important Note on Persistent Volumes

The **Persistent Volumes (PVs)** for Prometheus, Grafana, and Loki are configured with a **Retain** reclaim policy.

As a result, deleting the namespace does **not** delete the PVs. Instead, their status becomes `Released`.

### Reset PVs for Re-deployment

Before re-deploying the stack, you must **manually clear the claimRef** from the retained PVs.

### Option 1: Patch with `kubectl`

Run the following command to remove the `claimRef` from a released PV:

```bash
kubectl patch pv <pv-name> -p '{"spec":{"claimRef": null}}'
```

> Replace <pv-name> with the name of the Persistent Volume.
> 
> 
> Example:
> 

```bash
kubectl patch pv prometheus-pv -p '{"spec":{"claimRef": null}}'
```

### Option 2: Manually Edit the PV

1. List PVs:
    
    ```bash
    kubectl get pv
    ```
    
2. Edit a released PV:
    
    ```bash
    kubectl edit pv <pv-name>
    ```
    
3. In the editor, delete the `claimRef:` block under `.spec`, then save and exit.
