# 🧠 Monitoring & Logging Stack on k3s (Self-Managed)

## 📌 Overview

This document outlines the deployment, configuration, and management of the self-managed observability stack built on a lightweight Kubernetes distribution (k3s). The stack includes monitoring, logging, and alerting using open-source tools with persistent local storage.

---

## 📦 Stack Components

| Component           | Purpose                                 |
|---------------------|------------------------------------------|
| **Prometheus**       | Metrics collection and storage          |
| **Grafana**          | Metrics and log visualization           |
| **Alertmanager**     | Alert routing and notification          |
| **Loki**             | Log aggregation                         |
| **Promtail**         | Log shipping from Kubernetes nodes      |
| **Node Exporter**    | Host-level metrics                      |
| **Blackbox Exporter**| External endpoint probing               |
| **Custom Dashboards**| Pre-built JSON dashboards for Grafana   |
| **Local PVs**        | Persistent storage for stateful apps    |

---

### Prerequisites

1. **Helm Installation**
    
    Ensure that Helm is installed:
    
    ```bash
    helm version 
    ```
    
    If not installed, you can install it using:
    
    ```bash
    curl https://raw.githubusercontent.com/helm/helm/main/scripts/get-helm-3 | bash
    
    ```
    
2. **Monitoring Node Hostname**
    
    Confirm that a node exists with the hostname `monitoring`:
    
    ```bash
    kubectl get nodes -o wide | grep monitoring   
    ```
    
    This ensures that the monitoring workloads will target the correct node.

3. **EBS Attached & Mounted at `/monitoring`**

- A separate EBS volume is attached to the **monitoring node**.
- It is mounted at:
    
    ```
    /monitoring
    ```
    

4. **Subdirectories Created for Persistent Storage**

The following directories are created within `/monitoring`:

- `/monitoring/grafana-data`
- `/monitoring/prometheus-data`
- `/monitoring/loki-data`

These are used as host paths for the respective PersistentVolume configurations.

5. **Permissions Set**

Permissions are applied recursively:

```bash
chmod -R 777 /monitoring
```

This allows read-write-execute for the owner and group, and read-execute for others.

Make sure the pods (Grafana, Prometheus, Loki) are either:

- Running with a user/group that matches ownership of `/monitoring`, or
- You have configured `fsGroup` in the pod's `securityContext` so the group permissions (775) are effective.

---

**Validation Checklist**

Run these commands to verify the setup:

```bash
# Ensure mount exists
mount | grep /monitoring

# Ensure expected directories exist
ls -l /monitoring

# Check permissions
ls -ld /monitoring/*
```

Example output:

```bash
drwxrwxr-x 2 root root 4096 Jul 10 14:01 /monitoring/grafana-data
drwxrwxr-x 2 root root 4096 Jul 10 14:01 /monitoring/prometheus-data
drwxrwxr-x 2 root root 4096 Jul 10 14:01 /monitoring/loki-data
```

> Run in order, from the root of the repository

> Ensure your monitoring node is tainted and labeled:
> We need these to be ran so that we can control the monitoring components deployment node selection and avoid other deployment scheduling on monitoring node

```bash
kubectl label node monitoring node-role=monitoring
kubectl taint node monitoring monitoring-only=true:NoSchedule
```

> Run the following script to setup kube state metrics
```bash
##Run this in deploy directory
chmod +x kube-metrics-setup.sh
./kube-metrics-setup.sh
```

## 🚀 Installation Guide

```bash
# 1. Create Namespace
kubectl apply -f namespace/ns.yml

# 2. Setup Storage
kubectl apply -f storage-class/expandable-local-storageclass.yml
kubectl apply -f pv/

# 3. Deploy Monitoring Components
kubectl apply -f node-exporter/
kubectl apply -f blackbox-exporter/
kubectl apply -f alert-manager/
kubectl apply -f prometheus/

# 4. Deploy Logging Components
kubectl apply -f loki/
kubectl apply -f promtail/

# 5. Create Grafana dashboards configmaps
chmod +x ./grafana/dashboards/create-dashboards.sh
./grafana/dashboards/create-dashboards.sh

# 6. Deploy Grafana
kubectl apply -f grafana/pvc.yml
kubectl apply -f grafana/datasources/
kubectl apply -f grafana/dashboards/global-dashboard-definition.yml
kubectl apply -f grafana/manifest.yml
```

### Note
> Please make sure not to run the PVC files multiple times and also do not delete the PVCs
