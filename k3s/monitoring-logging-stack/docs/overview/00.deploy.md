## Prerequisites

- A Kubernetes cluster with access to the monitoring node.
- SSH access to the monitoring node using a private key.
- Git installed on your local machine.
- Kubectl configured to interact with your cluster.

---

## Deployment Steps

### 1. Clone the Repository

Clone the monitoring stack repository to your local machine:

```bash
git clone <your-repo-url>
cd <repo-name>
```

### 2. Checkout to the Deployment Branch

Make sure you are on the `deploy-v1` branch:

```bash
git checkout deploy-v1
```

### 3. Navigate to the Deploy Directory

Change to the `deploy` directory where the deployment scripts are located:

```bash
cd deploy
```

### 4. Run the Deployment Script

Execute the deployment script by providing:

- The IP address of the monitoring node.
- The path to your SSH private key for access.

```bash
./deploy.sh <ip-of-monitoring-node> <path-to-private-key>
```

> Example:
> 

```bash
./deploy 192.168.1.100 ~/keys/prod/app-server
```

This command will:

- Set up the required Kubernetes resources.
- Mount the EBS volume on /monitoring of the monitoring node server.
- Add required taints and label.
- Apply manifests for Prometheus, Grafana, Loki, AlertManager, etc.
- Configure persistent volumes and storage class.
- Deploy dashboards and exporters.
