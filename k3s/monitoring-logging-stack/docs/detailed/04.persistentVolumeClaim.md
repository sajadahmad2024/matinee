## 📦 3. PersistentVolumeClaims (PVCs)

The following PersistentVolumeClaims (PVCs) are used to bind to specific PersistentVolumes (PVs) to persist data for **Grafana**, **Prometheus**, and **Loki**.

These PVCs request specific storage amonts and explicitly bind to their corresponding static PVs using the `volumeName` field.

---

### 🔗 Binding Notes

- All PVCs use the `local-path-expandable` storage class, which supports **volume expansion**.
- The `volumeName` field ensures PVCs bind to **pre-created static PVs** (`grafana-pv`, `prometheus-pv`, `loki-data-pv`).
- Each PV must exist **before** applying these PVCs; otherwise, binding will fail.

---

### ✅ Checklist Before Applying

> Make sure the following are in place:
> 
- [x]  A separate EBS volume (or disk) is mounted on the **`monitoring` node** at `/monitoring`.
- [x]  These directories exist on that node:
    - `/monitoring/grafana-data`
    - `/monitoring/prometheus-data`
    - `/monitoring/loki-data`
- [x]  Correct permissions are set:

```bash
sudo mkdir -p /monitoring/{grafana-data,prometheus-data,loki-data}
sudo chmod -R 775 /monitoring
```
- [x]  Make sure all the PVs required are deployed

This ensures all components can read/write to their respective paths.
