## Promtail - Log Aggregation for the Cluster

### Overview

Promtail is deployed as a **DaemonSet** to collect logs from **all pods across all namespaces** and forward them to **Loki** for centralized logging and querying via Grafana.

### Configuration Details

- Configured via a `ConfigMap` named `promtail-config`
- Uses `kubernetes_sd_configs` to dynamically discover all running pods
- Relabeling is used to extract metadata such as `namespace`, `pod_name`, and `container_name`
- Log file paths are resolved using pod UID and container name
- Pushes logs to the internal `loki.monitoring.svc.cluster.local:3100` endpoint

### Key Features

- Collects logs from `/var/log/pods` on each node
- Deploys using a `DaemonSet`, ensuring Promtail runs on **every node**
- Includes a `ServiceAccount`, `ClusterRole`, and `ClusterRoleBinding` for necessary access
- Provides a `Service` on port `9080`, though Loki scrapes from Promtail's push endpoint

### Important Notes

- **Do not modify** this configuration. It is already set up to:
    - Collect logs from **all pods in all namespaces**
    - Send logs to the **correct internal Loki service URL**
    - Use appropriate relabeling and file path resolution
- Changing mount paths, config file structure, relabeling logic, or the Loki endpoint may **break log collection or indexing**
