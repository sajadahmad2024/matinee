## 🗄️ 2. PersistentVolumes (grafana-pv, loki-data-pv, prometheus-pv)

### 📌 Purpose

These `PersistentVolumes` are **manually provisioned hostPath volumes** designed to store data for Grafana dashboards, Loki logs, and Prometheus metrics respectively. By managing PVs manually:

- You have **greater control** over storage location and lifecycle.
- You ensure **data persists** across pod restarts or even re-creations.

---

### ⚙️ Prerequisite: Host Node Storage Setup

To ensure persistence and proper functioning of these components, follow the below setup on the **monitoring node** (the node labeled with `kubernetes.io/hostname=monitoring`):

1. **Mount a dedicated EBS volume** to the node at:
    
    ```bash
    /monitoring
    
    ```
    
    > 📌 This ensures that monitoring data survives node restarts or pod deletions.
    > 
2. **Create the necessary subdirectories** used by the PersistentVolumes:
    
    ```bash
    sudo mkdir -p /monitoring/{grafana-data,loki-data,prometheus-data}
    
    ```
    
3. **Set appropriate permissions** to ensure all monitoring-related pods (usually running under a non-root user) can read/write to these directories:
    
    ```bash
    sudo chmod -R 775 /monitoring
    
    ```
    
    If needed, you can also `chown` the directories to a specific user/group used by your monitoring pods. But `775` generally works well with Kubernetes pod security context defaults.
    

---

### 📁 Directory Summary

| Directory Path | Used By | PV Name | Size |
| --- | --- | --- | --- |
| `/monitoring/grafana-data` | Grafana | `grafana-pv` | 2Gi |
| `/monitoring/loki-data` | Loki | `loki-data-pv` | 15Gi |
| `/monitoring/prometheus-data` | Prometheus | `prometheus-pv` | 10Gi |

### ⚙️ General PV Settings

| Field | Value |
| --- | --- |
| `accessModes` | `ReadWriteOnce` (RWO) |
| `persistentVolumeReclaimPolicy` | `Retain` (recommended ✅) |
| `storageClassName` | `local-path-expandable` |
| `hostPath.path` | Custom local directory under `/monitoring/...` |
| `nodeAffinity` | Restricts the PV to a specific node (named `monitoring`) |

---

### 📄 Detailed PV Definitions

---

### 🔹 1. `grafana-pv`

```yaml

apiVersion: v1
kind: PersistentVolume
metadata:
  name: grafana-pv
spec:
  capacity:
    storage: 2Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  storageClassName: local-path-expandable
  hostPath:
    path: /monitoring/grafana-data
    type: Directory
  nodeAffinity:
    required:
      nodeSelectorTerms:
        - matchExpressions:
            - key: kubernetes.io/hostname
              operator: In
              values:
                - monitoring

```

- **Purpose**: Stores Grafana dashboards, plugins, and user data.
- **Size**: 2Gi — sufficient unless large dashboard history is retained.
- **Host path**: `/monitoring/grafana-data`
- **Affinity**: This volume can be bound only on the node named `monitoring`.

### 🔹 2. `loki-data-pv`

```yaml

apiVersion: v1
kind: PersistentVolume
metadata:
  name: loki-data-pv
  labels:
    loki-pv: "true"
spec:
  capacity:
    storage: 15Gi
  accessModes:
    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  storageClassName: local-path-expandable
  hostPath:
    path: /monitoring/loki-data
    type: Directory
  nodeAffinity:
    required:
      nodeSelectorTerms:
        - matchExpressions:
            - key: kubernetes.io/hostname
              operator: In
              values:
                - monitoring

```

- **Purpose**: Persists Loki's indexed logs and chunks.
- **Size**: 15Gi — sized to hold a moderate amount of log data.
- **Host path**: `/monitoring/loki-data`

---

    - ReadWriteOnce
  persistentVolumeReclaimPolicy: Retain
  storageClassName: local-path-expandable

    path: /monitoring/prometheus-data

      nodeSelectorTerms:

            - key: kubernetes.io/hostname
              operator: In
              values:
                - monitoring

- **Purpose**: Stores time-series metrics, rules, and WAL (Write Ahead Logs).
- **Size**: 10Gi — adjustable depending on data retention needs.
- **Host path**: `/monitoring/prometheus-data`
